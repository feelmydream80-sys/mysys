# 데이터 분석 화면 설명

## 1. 개요

데이터 분석 화면은 MSYS 시스템의 수집 및 가공 데이터를 상세하게 분석하고 시각화하는 기능을 제공합니다. 날짜 범위, Job ID, 장애코드 등의 필터를 통해 특정 조건의 데이터를 조회할 수 있으며, 데이터의 추이 분석, 통계 요약, AI 기반 인사이트를 제공합니다.

## 2. 화면 구성 요소 상세 설명

### 2.1 헤더/탭 영역
- **페이지 제목**: "데이터분석" (한글 표시)
- **접기/펴기 기능**: collapsible_controls.html를 통한 카드 접기/펴기
- **관리자 권한 표시**: `is_admin` 플래그에 따른 추가 기능 노출

### 2.2 메인 컨텐츠 영역

#### 필터 영역 (접기/펴기 가능)
| 필터 요소 | 유형 | 필수/선택 | 유효성 검증 | 기본값 | 설명 |
|----------|------|----------|------------|--------|------|
| 시작일 | date input | 필수 | YYYY-MM-DD 형식 | 오늘-30일 | 조회 시작 날짜 |
| 종료일 | date input | 필수 | YYYY-MM-DD 형식 | 오늘 | 조회 종료 날짜 |
| Job ID | select | 선택 | 시스템 등록 Job ID | 전체 | 특정 Job으로 필터링 |
| 장애코드 | select | 선택 | 시스템 정의 코드 | 전체 | 특정 오류로 필터링 |

**필터 조합별 조회 결과 표:**

| 필터 조합 | 조회 범위 | 결과 데이터 | 사용 사례 |
|----------|----------|------------|----------|
| 날짜만 | 선택 기간 전체 | 모든 Job의 모든 데이터 | 전체 현황 파악 |

| 날짜 + 장애코드 | 특정 오류의 기간 데이터 | 오류별 패턴 분석 | 장애 트렌드 파악 |
| 전체 필터 | 특정 Job의 특정 오류 | 정밀한 문제 진단 | 특정 이슈 심층 분석 |

#### 요약/통계 카드 영역

|----------|--------|----------|------|------|

## 3. 데이터 구조 및 의미

### 3.1 표시되는 데이터 필드 설명

#### 수집시간(hr) 필드
- **데이터 유형**: DECIMAL(10,2)
- **단위**: 시간(hr)
- **의미**: 데이터 수집 작업의 실제 실행 시간
- **계산 방식**: `end_dt - start_dt` (타임스탬프 차이)
- **정상 범위**: 0.01 ~ 24.00 시간

#### 예측 수집시간(hr) 필드
- **데이터 유형**: DECIMAL(10,2)
- **단위**: 시간(hr)
- **의미**: 과거 데이터 기반으로 예측한 수집 시간
- **계산 방식**: 최근 3회 평균 × 1.1 (안전 마진 10%)
- **사용 목적**: 예상 완료 시간 표시

#### 완전성(%) 필드
- **데이터 유형**: DECIMAL(5,2)
- **단위**: 백분율(%)
- **의미**: 수집된 데이터의 완전도
- **계산 방식**: `(실제 수집 건수 ÷ 계획 수집 건수) × 100`
- **정상 범위**: 95% ~ 100%

### 3.2 데이터 계산/집계 방식

#### 평균 소요시간 계산
- **실시간 vs 배치**: 실시간 계산 (필터 변경 시 즉시)
- **데이터 업데이트 주기**: 필터 변경 이벤트 발생 시
- **캐싱 전략**: 계산 결과 5분 캐시
- **이상치 처리**: 최대/최소 값 제외 옵션

#### 성공률 변화 계산
- **전일 대비 계산**: `(금일 성공률 - 전일 성공률) ÷ 전일 성공률 × 100`
- **데이터 소스**: 일별 집계 데이터
- **표시 형식**: ±X.X% (소수점 1자리)
- **색상 표시**: 빨강(감소), 파랑(증가), 회색(변화없음)

### 3.3 상태 표시 기준 및 색상 코드

#### 데이터 행 상태 표시
- **정상 상태**: 배경색 흰색, 텍스트 검정
  - 조건식: `완전성 >= 95% AND 연속실패 < 3`
- **경고 상태**: 배경색 노랑 (#FFF3CD), 텍스트 검정
  - 조건식: `(완전성 < 95% AND 완전성 >= 80%) OR (연속실패 >= 3 AND 연속실패 < 5)`
- **위험 상태**: 배경색 빨강 (#F8D7DA), 텍스트 검정
  - 조건식: `완전성 < 80% OR 연속실패 >= 5 OR status = 'CD902'`

#### 수치 필드 색상 표시
- **평균보다 빠름**: 파랑 (#007BFF)
  - 조건식: `수집시간 < 평균소요시간 × 0.8`
- **평균보다 느림**: 주황 (#FD7E14)
  - 조건식: `수집시간 > 평균소요시간 × 1.2`
- **이상치**: 빨강 (#DC3545)
  - 조건식: `이상치 = true`

**임계값 출처**: 해당 값들은 관리자 설정 → 데이터 분석 설정에서 변경 가능하며 현재 값들은 시스템 기본값을 기준으로 작성되었다.

## 4. 사용자 인터랙션

### 4.1 기본 사용 방법 (단계별 절차)

1. **필터 설정**
   - 시작일/종료일 선택 (기본값: 최근 30일)
   - Job ID 선택 (옵션: 특정 Job 또는 전체)
   - 장애코드 선택 (옵션: 특정 오류 또는 전체)
   - 목표: 분석할 데이터 범위 지정
   - 결과 확인: 필터 아래 데이터 자동 갱신

2. **요약 정보 확인**
   - 최대/최소 소요시간 카드 확인
   - 실패 호출 건수, 중단 횟수 확인
   - 목표: 전체 데이터 현황 파악
   - 결과 확인: 수치와 색상으로 상태 파악

3. **Job ID 상세정보 조회**
   - 테이블 스크롤 또는 검색 사용
   - 특정 Job의 주기와 설명 확인
   - 목표: Job별 기본 정보 파악
   - 결과 확인: cron 표현식과 설명 표시

4. **상세 데이터 분석**
   - 수집 및 가공 데이터 테이블 확인
   - 수집시간, 완전성, 연속실패 등 분석
   - 목표: 데이터 수집 성능 상세 분석
   - 결과 확인: 색상 코딩으로 이상치 식별

### 4.2 고급 기능 사용법

- **데이터 유지**: 페이징 시 필터 조건 유지

- **CSV 내보내기**: 테이블 우측 상단 버튼

## 5. 문제 해결 및 참고사항

### 5.1 일반적인 오류 상황 및 해결 방법

#### 데이터 조회 실패: "API 호출 오류"
- **증상**: 테이블에 "데이터 조회 실패" 메시지 표시
- **원인**: 네트워크 문제, 서버 오류, 세션 만료
- **해결 절차**:
  1. 인터넷 연결 확인
  2. 브라우저 새로고침 (F5)
  3. 로그인 상태 확인 및 재로그인
- **예방 조치**: 안정적인 네트워크 환경 사용

#### 필터 적용 안됨: "필터가 반영되지 않음"
- **증상**: 필터 변경 후 데이터가 업데이트되지 않음
- **원인**: JavaScript 오류, 이벤트 핸들러 문제
- **해결 절차**:
  1. 브라우저 콘솔 오류 확인 (F12)
  2. 페이지 새로고침
  3. 다른 브라우저 시도
- **예방 조치**: 최신 브라우저 사용

#### 검색 결과 없음: "No data found"
- **증상**: 검색 후 결과가 전혀 표시되지 않음
- **원인**: 검색어 오타, 필터 조합 문제
- **해결 절차**:
  1. 검색어 철자 확인
  2. 필터 조건 완화 (전체 선택)
  3. 데이터 존재 기간 확인
- **예방 조치**: 검색어 정확히 입력

### 5.2 데이터 이상 시 확인사항

#### 수집시간 0 또는 음수 값
- **확인사항**: 수집시간(hr) 컬럼에 0.00 또는 음수 표시
- **조치**: 해당 Job의 로그 확인, 수집 프로세스 검증
- **원인**: 타임스탬프 기록 오류, 시스템 시간 동기화 문제

#### 완전성 0% 데이터
- **확인사항**: 완전성(%)가 0.00으로 표시
- **조치**: Job 설정 확인, 데이터 소스 연결 검증
- **원인**: 네트워크 단절, 권한 문제, 데이터 소스 오류

#### 연속실패 10회 이상
- **확인사항**: 연속실패 컬럼에 10 이상의 높은 숫자
- **조치**: 즉시 Job 중단 및 관리자 알림
- **원인**: 시스템 장애, 네트워크 문제, 데이터 소스 변경

### 5.3 성능 저하 시 조치 방법

#### 페이지 로딩 지연 (10초 이상)
- **확인사항**: 필터 변경 후 데이터 로딩이 오래 걸림
- **조치**: 데이터 기간 축소, Job ID 특정 선택
- **원인**: 대량 데이터 조회, 서버 부하

#### 검색 응답 지연
- **확인사항**: 검색어 입력 후 결과 표시까지 지연
- **조치**: 검색어 길이 줄이기, 더 구체적인 검색어 사용
- **원인**: 전체 데이터 스캔, 복잡한 검색 조건

#### 메모리 부족 경고
- **확인사항**: 브라우저 메모리 부족 메시지
- **조치**: 브라우저 재시작, 데이터 범위 축소
- **원인**: 대량 데이터 로딩, 브라우저 캐시 누적

### 5.4 관련 메뉴 및 참고 자료

#### 관련 메뉴
- **대시보드**: 실시간 현황 및 요약 정보
- **차트 분석**: 데이터 시각화 및 추이 분석

- **데이터 명세서**: Job별 상세 설정 정보

#### 참고 자료

- **데이터베이스 스키마**: tb_con_hist, tb_con_mst 테이블 구조

```python
# AnalysisService.get_raw_data_with_stats() - 데이터 가공
class AnalysisService:
    def get_raw_data_with_stats(self, start_date, end_date, job_ids=None, all_data=False):
        # 1. 원본 데이터 조회
        raw_data = self.analytics_dao.get_raw_data(start_date, end_date, job_ids, all_data)

        # 2. 통계 계산 추가
        for item in raw_data:
            item['avg_duration'] = self._calculate_avg_duration(item['job_id'])
            item['fail_streak'] = self._calculate_fail_streak(item['job_id'])
            item['is_outlier'] = self._detect_outlier(item['duration'], item['avg_duration'])

        # 3. 정렬 및 필터링
        processed_data = self._apply_sorting_and_filters(raw_data)

        return processed_data
```

export async function fetchAndRenderAll() {
    const filters = getCurrentFilters();

    try {
        // 병렬 데이터 조회
        const [summary, trend, raw] = await Promise.all([
            fetchSummaryData(filters),
            fetchTrendData(filters),
            fetchRawData(filters)

        renderRawTable(getRawData());
        renderTrendChart(trend);

    } catch (error) {
        showError('데이터 조회 실패: ' + error.message);
    }
}

// 실시간 검색 구현
export function initializeSearch() {
    const searchInput = document.getElementById('rawDataSearch');

    searchInput.addEventListener('input', (event) => {
        const searchTerm = event.target.value.toLowerCase();
        const filteredData = getRawData().filter(item =>
            item.job_id.toLowerCase().includes(searchTerm) ||
            item.status.toLowerCase().includes(searchTerm)
        );

        updatePaginationData('rawPagination', filteredData);
        renderRawTable(filteredData);
    });
