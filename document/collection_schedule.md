# 데이터 수집 일정 표시 오류 해결 보고서

## 1. 문제 현황

데이터 수집 일정 화면에서 정상적으로 수집이 완료된 작업들이 '미수집' 상태로 잘못 표시되는 문제가 발생했습니다.

## 2. 원인 분석

문제의 원인은 다음과 같이 여러 단계에 걸쳐 파악되었습니다.

1.  **1차 원인: 부정확한 시간 비교 로직**
    *   `service/collection_schedule_service.py` 파일에서 하루에 한 번 실행되는 작업의 성공 여부를 판단할 때, 예정된 시간과 실제 수집 시간이 정확히 일치해야만 성공으로 처리했습니다. 이로 인해 약간의 시간 오차만 발생해도 '미수집'으로 잘못 표시되었습니다.

2.  **2차 원인: 수집 기록 데이터 누락**
    *   1차 원인 수정 후에도 문제가 해결되지 않아 로그를 추가하여 확인한 결과, 데이터베이스에서 실제 수집 기록을 가져오지 못해 비교 로직이 실행되지 않았습니다.

3.  **3차 원인: 시간 정보 손실**
    *   수집 기록을 조회하는 `sql/dashboard/dashboard_sql.py` 파일의 쿼리에서 `collection_date`를 `date` 타입으로 형변환하면서 시간 정보가 손실되었습니다. 이로 인해 시간 비교가 불가능했습니다.

4.  **최종 원인: 날짜 비교 누락**
    *   시간 정보 손실 문제를 해결한 후에도 일부 작업이 여전히 '미수집'으로 표시되었습니다. 로그를 통해 확인한 결과, 예정된 날짜와 실제 수집 날짜가 다른데도 시간을 비교하여 잘못된 결과를 반환하고 있었습니다.
5.  **숨겨진 문제: 자정 경계에서의 오류**
    *   기존 날짜 비교 로직은 자정(`00:00`)을 기준으로 날짜가 변경될 때 문제를 일으킬 수 있었습니다. 예를 들어, `23:58`에 예정된 작업이 네트워크 지연 등으로 `00:02`에 실행되면 날짜가 달라져 '미수집'으로 잘못 처리될 가능성이 있었습니다.

## 3. 해결 방안

각 단계별 원인을 해결하기 위해 다음과 같이 조치했습니다.

1.  **1단계: 시간 비교 로직 수정**
    *   모든 데이터 수집 작업에 대해 예정된 시간을 기준으로 **5분의 허용 시간**을 두어, 이 시간 내에 수집이 시작되었다면 성공으로 처리하도록 로직을 통일했습니다.

2.  **2단계: 데이터 조회 함수 변경**
    *   기존 기능에 영향을 주지 않으면서 시간 정보(`start_dt`)를 포함하여 수집 기록을 조회하는 `get_collection_history_for_schedule_with_start_dt` 함수를 사용하도록 `service/collection_schedule_service.py`를 수정했습니다.

3.  **3단계: 최종 로직 수정**
    *   `utils/datetime_utils.py`의 `is_within_schedule_grace_period` 함수에서 **날짜 비교 로직을 제거**하고, 두 시간의 절대적인 차이만을 비교하여 자정 경계에서 발생하는 문제를 해결했습니다.

4.  **4단계: 매칭 로직 개선**
    *   스케줄과 히스토리의 매칭 방식을 **날짜별 순차 매칭**으로 변경했습니다. 각 날짜별로 스케줄을 시간순 정렬하고, 해당 날짜의 실행 기록을 시간순 정렬하여 순차적으로 매칭합니다. 이로써 스케줄 외 실행이 과거 날짜의 스케줄에 잘못 매칭되는 문제를 해결했습니다.

## 4. 결론

위와 같은 조치를 통해 데이터 수집 일정의 성공 여부 판단 로직을 수정하여 문제를 해결했습니다. 이제 모든 수집 작업의 상태가 정확하게 표시됩니다.
