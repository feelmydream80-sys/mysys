## 1. 개요

본 문서는 Flask 기반 데이터 수집 모니터링 시스템(MSYS)의 코드 품질 향상 및 유지보수성 강화를 위한 리팩토링 계획을 수립합니다. 현재 시스템이 정상 동작하는 상태에서 테스트 인프라를 구축하고, 단계적으로 리팩토링을 진행하여 기능 안정성을 보장합니다.

### 1.1. 프로젝트 배경
- **시스템 규모**: Python Flask 웹 애플리케이션, 429개 파일, 62,539줄 코드
- **주요 기능**: 대시보드, 데이터 분석, 차트 분석, 사용자 인증, 데이터 수집 모니터링
- **기술 스택**: Flask, MySQL, HTML/CSS/JavaScript, Chart.js

### 1.2. 리팩토링 목표
- 코드 가독성 및 유지보수성 향상
- 아키텍처 일관성 확보
- 테스트 커버리지 확대를 통한 안정성 보장
- 기술 부채 감소

## 3. 발견된 리팩토링 기회

### 3.1. 구조적 문제점

#### 3.1.1. 라우트 구조 불일관성
- **문제**: `routes/analysis_routes.py` 등에서 UI 렌더링과 API 응답이 혼재
- **영향**: 유지보수 어려움, 관심사 분리 원칙 위반
- **해결**: `routes/ui/`와 `routes/api/`로 완전 분리

#### 3.1.2. 메인 애플리케이션 파일 과도한 책임
- **문제**: `msys_app.py`의 `create_app()` 함수가 300줄 이상
- **문제점**: 로깅, 인증, DB 초기화가 한 곳에 집중
- **해결**: 관심사별 모듈 분리

#### 3.1.3. 코드 중복
- **문제**: 시간대 변환(KST) 로직이 여러 라우트에서 반복
- **해결**: 공통 유틸리티 함수로 추출

### 3.2. 코드 품질 문제점

#### 3.2.1. 긴 함수 및 클래스
- **문제**: 일부 서비스 메서드가 복잡하고 길어 단일 책임 원칙 위반
- **해결**: 작은 메서드로 분해

#### 3.2.2. 설정 관리 분산
- **문제**: `.env`와 하드코딩 값이 혼재
- **해결**: 중앙화된 설정 관리

#### 3.2.3. 네이밍 불일관성
- **문제**: snake_case와 camelCase 혼용
- **해결**: 프로젝트 전반 네이밍 컨벤션 적용

## 4. 테스트 전략

### 4.1. 테스트 레벨 분류

#### 4.1.1. 단위 테스트 (Unit Tests)
- **대상**: 서비스 클래스, 유틸리티 함수, DAO 로직
- **목적**: 개별 컴포넌트의 올바른 동작 검증
- **예시**: `AnalysisService.get_dynamic_chart_data()`

#### 4.1.2. 통합 테스트 (Integration Tests)
- **대상**: API 엔드포인트, 데이터베이스 연동
- **목적**: 컴포넌트 간 상호작용 검증
- **예시**: `/api/analysis/dynamic-chart` 엔드포인트

#### 4.1.3. E2E 테스트 (End-to-End Tests)
- **대상**: 전체 사용자 플로우
- **목적**: 실제 사용자 시나리오 검증
- **예시**: 로그인 → 대시보드 접근 → 데이터 조회

### 4.2. 테스트 인프라 구축

#### 4.2.1. 기존 인프라 활용
- `conftest.py`의 fixture들 유지 및 확장
- 테스트용 DB 연결 및 트랜잭션 관리

#### 4.2.2. 신규 테스트 파일 구조
```
tests/
├── __init__.py
├── conftest.py                    # 기존 유지
├── test_api_analysis.py          # 기존 확장
├── test_api_auth.py              # 신규
├── test_api_dashboard.py         # 신규
├── test_api_card_summary.py      # 신규
├── service/
│   ├── test_analysis_service.py  # 신규
│   ├── test_dashboard_service.py # 신규
│   └── test_user_service.py      # 기존
└── utils/
    └── test_datetime_utils.py    # 신규
```

#### 4.2.3. 테스트 실행 및 모니터링
- **실행**: `pytest` 또는 `pytest -v`
- **커버리지**: `pytest --cov=. --cov-report=html`
- **CI/CD**: 자동화된 테스트 실행
