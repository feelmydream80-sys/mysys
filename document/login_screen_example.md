# 로그인 화면 설명

## 1. 개요

로그인 화면은 MSYS 시스템에 접근하기 위한 사용자 인증 화면입니다. 사용자 ID와 비밀번호를 입력받아 시스템 접근 권한을 검증하며, 인증 성공 시 대시보드로 자동 이동합니다.

## 2. 화면 구성 요소 상세 설명

### 2.1 헤더/탭 영역

## 3. 데이터 구조 및 의미

### 3.1 표시되는 데이터 필드 설명

#### 사용자 ID 필드
- **데이터 유형**: VARCHAR(50)
- **단위**: 텍스트
- **의미**: 시스템에 등록된 고유 사용자 식별자
- **계산 방식**: 별도 계산 없음, 직접 입력 값 사용

#### 비밀번호 필드
- **데이터 유형**: PASSWORD (마스킹 처리)
- **단위**: 텍스트
- **의미**: 사용자 인증을 위한 비밀번호
- **계산 방식**: 별도 계산 없음, SHA-256 해시 변환 후 검증

### 3.2 데이터 계산/집계 방식

#### 인증 검증 방식
- **실시간 vs 배치**: 실시간 인증 (즉시 응답)
- **데이터 업데이트 주기**: 입력 시 즉시 검증
- **캐싱 전략**: 인증 토큰 24시간 유효

#### 세션 관리
- **세션 생성**: 인증 성공 시 JWT 토큰 발급
- **세션 유지**: 30분 자동 연장
- **세션 만료**: 24시간 후 자동 로그아웃

### 3.3 상태 표시 기준 및 색상 코드

#### 입력 필드 상태
- **정상 상태**: 테두리 색상 #E0E0E0 (회색)
  - 조건식: `필드_값 != 빈_문자열 AND 유효성_검증_통과`
- **오류 상태**: 테두리 색상 #FF6B6B (빨강)
  - 조건식: `필드_값 == 빈_문자열 OR 유효성_검증_실패`
- **포커스 상태**: 테두리 색상 #4A90E2 (파랑)
  - 조건식: `필드_포커스_활성화`

#### 버튼 상태
- **활성화 상태**: 배경색 #007BFF (파랑), 텍스트 #FFFFFF (흰색)
  - 조건식: `모든_필수_필드_입력_완료 AND 유효성_검증_통과`
- **비활성화 상태**: 배경색 #CCCCCC (회색), 텍스트 #666666
  - 조건식: `필수_필드_미입력 OR 유효성_검증_실패`

**임계값 출처**: 해당 값들은 관리자 설정에 의해 변경 가능하며 현재 값들은 기본값을 기준으로 작성되었다.

## 4. 사용자 인터랙션

### 4.1 기본 사용 방법 (단계별 절차)

1. **페이지 접근**
   - 브라우저에서 시스템 URL 입력
   - 목표: 로그인 화면 표시
   - 결과 확인: 탭 메뉴와 로그인 폼 표시

2. **사용자 ID 입력**
   - ID 입력 필드 클릭
   - 영문/숫자 조합으로 ID 입력
   - 목표: 유효한 사용자 ID 입력
   - 결과 확인: 필드 테두리 파랑으로 변경

3. **비밀번호 입력**
   - 비밀번호 필드 클릭
   - 보안 정책 준수 비밀번호 입력
   - 목표: 올바른 비밀번호 입력
   - 결과 확인: 마스킹된 별표(*) 표시

4. **로그인 실행**
   - 로그인 버튼 클릭
   - 목표: 시스템 인증 및 접근
   - 결과 확인: 대시보드 페이지로 이동

### 4.2 고급 기능 사용법

#### 브라우저 자동 완성
- 이전 로그인 정보 자동 표시
- 보안 정책에 따라 제한될 수 있음

#### 키보드 단축키
- **Tab**: 다음 필드로 이동
- **Enter**: 로그인 버튼 클릭 (활성화 상태일 때)
- **Esc**: 입력 필드 초기화

### 4.3 필터 및 검색 기능

### 4.4 데이터 내보내기/가져오기

## 5. 문제 해결 및 참고사항

### 5.1 일반적인 오류 상황 및 해결 방법

#### 오류 유형별 대응 표

| 오류 유형 | HTTP 상태 코드 | 증상 | 즉시 조치 | 추가 확인사항 |
|----------|---------------|------|----------|--------------|
| 사용자 없음 | 404 | "사용자를 찾을 수 없습니다" | ID 재입력 | 대소문자 확인 |
| 비밀번호 불일치 | 401 | "비밀번호가 올바르지 않습니다" | Caps Lock 확인 | 남은 시도 횟수 확인 |
| 계정 잠금 | 423 | "계정이 잠금되었습니다" | 30분 대기 | 잠금 해제 시간 확인 |
| 서버 오류 | 500 | "서버 내부 오류" | 잠시 후 재시도 | 네트워크 상태 확인 |
| 네트워크 오류 | - | "연결 실패" | 네트워크 확인 | 프록시 설정 확인 |

#### 오류별 해결 절차 표

| 오류 케이스 | 1단계 조치 | 2단계 조치 | 3단계 조치 | 예상 소요시간 |
|------------|------------|------------|------------|--------------|
| **사용자 없음** | ID 철자 확인 | 대소문자 확인 | 관리자 문의 | 즉시 ~ 5분 |
| **비밀번호 오류** | Caps Lock 확인 | 비밀번호 재입력 | 초기화 요청 | 즉시 ~ 10분 |
| **계정 잠금** | 잠금 시간 확인 | 30분 대기 | 관리자 문의 | 30분 ~ 1시간 |
| **서버 오류** | 1분 대기 후 재시도 | 브라우저 새로고침 | 관리자 문의 | 1분 ~ 30분 |
| **네트워크 오류** | WiFi 재연결 | VPN 확인 | IT 지원 요청 | 즉시 ~ 15분 |

#### 예방 조치 가이드라인 표

| 위험 요인 | 예방 방법 | 빈도 | 담당 |
|----------|----------|------|------|
| 오타 입력 | 자동 완성 활용 | 상시 | 사용자 |
| Caps Lock | 상태 표시 아이콘 | 상시 | 시스템 |
| 비밀번호 공유 | 교육 및 정책 | 분기별 | 관리자 |
| 동시 접속 | 세션 관리 | 실시간 | 시스템 |
| 브라우저 캐시 | 정기 정리 권장 | 월별 | 사용자 |

### 5.2 데이터 이상 시 확인사항

#### 세션 만료
- **확인사항**: 갑작스러운 로그아웃
- **조치**: 재로그인
- **원인**: 24시간 세션 만료 또는 동시 접속

#### 브라우저 캐시 문제
- **확인사항**: 로그인 불가, 페이지 로드 지연
- **조치**: 브라우저 캐시 삭제 및 새로고침
- **원인**: 쿠키/세션 데이터 손상

### 5.3 성능 저하 시 조치 방법

#### 로그인 응답 지연
- **확인사항**: 버튼 클릭 후 응답 없음
- **조치**: 네트워크 연결 확인, 서버 상태 점검
- **원인**: 네트워크 문제, 서버 과부하

#### 페이지 로드 지연
- **확인사항**: 화면 표시까지 오래 걸림
- **조치**: 브라우저 캐시 정리, 네트워크 속도 확인
- **원인**: 대용량 리소스, 느린 인터넷

### 5.4 관련 메뉴 및 참고 자료

#### 관련 메뉴
- **회원가입 화면**: 신규 사용자 등록
- **비밀번호 변경**: 기존 비밀번호 수정
- **대시보드**: 로그인 성공 후 이동 화면

#### 참고 자료
- **사용자 매뉴얼**: 시스템 전반적 사용법
- **보안 정책**: 비밀번호 정책 및 계정 관리
- **FAQ**: 자주 묻는 질문과 답변

```python
# AuthService.authenticate_user() - 4단계 처리
class AuthService:
    def authenticate_user(self, user_id: str, password: str):
        # 1. 사용자 조회 및 존재 확인
        user = self.auth_dao.get_user_by_id(user_id)

        # 2. 계정 상태 검증 (승인 여부, 잠금 상태)
        self._validate_account_status(user)

        # 3. 비밀번호 검증 (해시 비교)
        self._verify_password(password, user['user_pwd'])

        # 4. 로그인 기록 및 세션 생성
        session_token = self._create_session(user)

        return {'success': True, 'token': session_token, 'user': user}
```

```javascript
// 데이터 관리 함수
export async function authenticateUser(credentials) {
    try {
        const response = await fetch('/api/auth/login', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(credentials)
        });

        if (response.ok) {
            const result = await response.json();
            // 세션 토큰 저장 및 리다이렉션
            localStorage.setItem('authToken', result.token);
            window.location.href = '/dashboard';
        } else {
            // 오류 메시지 표시
            showLoginError(response.status === 401 ? '인증 실패' : '서버 오류');
        }
    } catch (error) {
        showLoginError('네트워크 오류');
    }
}
```
