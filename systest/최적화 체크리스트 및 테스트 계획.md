## 2. 최적화 체크리스트

### 2.1 단계 1: 공통 모듈화 (우선순위 1)

#### 2.1.1 데코레이터 통합
- [ ] `auth_routes.py`의 모든 권한 데코레이터를 `utils/decorators.py`로 추출
- [ ] `@permission_required` 데코레이터 추가 (권한별로 사용 가능)
- [ ] 모든 라우트에서 기존 데코레이터를 새 데코레이터로 교체

#### 2.1.2 DB 연결 관리
- [ ] `utils/db.py`에 컨텍스트 매니저 기반의 `db_connection()` 함수 추가
- [ ] 모든 API에서 직접 `get_db_connection()` 호출을 `with db_connection() as conn:`으로 교체
- [ ] 트랜잭션 관리 개선 (커밋·롤백 자동화)

#### 2.1.3 로깅 유틸리티
- [ ] `utils/logging_config.py`에 API 로깅 데코레이터 `@log_api_request` 추가
- [ ] 각 API 함수에 로깅 데코레이터 적용
- [ ] 로그 레벨·포맷 일관성 유지

### 2.2 단계 2: 서비스 계층 개선 (우선순위 2)

#### 2.2.1 의존성 주입
- [ ] 모든 서비스 클래스에 DI 패턴 적용
- [ ] `MngrSettService.__init__()`에서 로컬 import 제거
- [ ] 서비스 초기화 시 필요한 의존성 주입

#### 2.2.2 설정 관리
- [ ] `DEFAULT_ADMIN_SETTINGS`를 `config.py`로 분리
- [ ] 서비스에서 하드코딩된 설정값을 config에서 가져오도록 수정

#### 2.2.3 트랜잭션 관리
- [ ] `utils/transaction.py`에 중앙 트랜잭션 관리자 추가
- [ ] 서비스 계층에서 트랜잭션 제어 개선

### 2.3 단계 3: 중복 코드 제거 (우선순위 3)

#### 2.3.1 사용자 권한 업데이트
- [ ] `mngr_sett_routes.py`에서 `save_all_settings()`와 `update_bulk_user_permissions()`의 중복 로직 통합
- [ ] `service/user_service.py`에 `update_user_permissions()` 메서드 추가 (단일/대량 업데이트 지원)

#### 2.3.2 데이터 조회 로직

- [ ] 공통 데이터 조회 메서드 추출 (예: `get_distinct_job_ids`)

### 2.4 단계 4: 코드 품질 향상 (우선순위 4)
