# 테스트 자동화 계획 (RPA 스타일)

## 1. 개요
테스트 자동화를 통해 사람이 직접 동작하는 것처럼 정상적인 동작을 기록하고 재현할 수 있는 방법을 제공합니다. 이 방법은 Selenium을 사용하여 웹 브라우저를 자동화하고, 테스트 시나리오를 스크립트로 작성하여 반복 실행할 수 있습니다.

## 2. 테스트 자동화 도구

### 2.1 주요 도구
- **Selenium WebDriver**: 웹 브라우저 자동화
- **Python 언어**: 테스트 스크립트 작성
- **pytest**: 테스트 프레임워크 (실행·보고서)
- **Allure**: 테스트 결과 보고서 생성
- **Docker**: 테스트 환경 격리

### 2.2 도구 설치
```bash
pip install selenium pytest allure-pytest
```


### 3.2 Page Object Pattern
각 페이지를 클래스로 구현하여 요소 찾기·작업 실행을 캡슐화합니다.

```python
# tests/pages/login_page.py
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

class LoginPage:
    def __init__(self, driver):
        self.driver = driver
        self.url = "http://localhost:5000/login"
        self.user_id_input = (By.ID, "user_id")
        self.password_input = (By.ID, "password")
        self.login_button = (By.CSS_SELECTOR, "button[type='submit']")
    
    def navigate(self):
        self.driver.get(self.url)
    
    def login(self, user_id, password):
        self.driver.find_element(*self.user_id_input).send_keys(user_id)
        self.driver.find_element(*self.password_input).send_keys(password)
        self.driver.find_element(*self.login_button).click()
    
    def is_login_successful(self):
        try:
            WebDriverWait(self.driver, 5).until(
                EC.presence_of_element_located((By.CSS_SELECTOR, ".dashboard"))
            )
            return True
        except:
            return False
```

## 4. 테스트 시나리오

### 4.1 일반 유저 시나리오

#### 4.1.1 로그인·로그아웃
```python
# tests/scenarios/test_general_user.py
import pytest
from tests.pages.login_page import LoginPage
from tests.pages.dashboard_page import DashboardPage

def test_login_and_logout(driver):
    login_page = LoginPage(driver)
    login_page.navigate()
    login_page.login("test_user", "test_password")
    
    dashboard_page = DashboardPage(driver)
    assert dashboard_page.is_dashboard_loaded()
    
    dashboard_page.logout()
    assert login_page.is_login_page_loaded()
```

## 5. 테스트 실행

### 5.1 테스트 실행 명령
```bash
# 모든 테스트 실행
pytest tests/ -v

# 특정 시나리오 실행
pytest tests/scenarios/test_general_user.py -v

# 브라우저 선택 (Chrome 기본)
pytest tests/ -v --browser=firefox

# Allure 보고서 생성
pytest tests/ -v --alluredir=reports/
allure serve reports/
```

### 5.2 테스트 실행 옵션
- **--browser**: 사용할 브라우저 (chrome, firefox, edge)
- **--headless**: headless 모드로 실행 (브라우저 창이 보이지 않음)
- **--slowmo**: 동작 속도 조절 (ms 단위)
- **--alluredir**: Allure 보고서 저장 경로

### 7.2 Slack·이메일 통보
테스트 실행 결과를 Slack이나 이메일로 통보합니다.

```python
# tests/utils/report_notifier.py
import requests
import smtplib
from email.mime.text import MIMEText

def send_slack_notification(result):
    webhook_url = "https://hooks.slack.com/services/..."
    payload = {
        "text": f"테스트 실행 완료\n성공: {result['passed']}, 실패: {result['failed']}, 스킵: {result['skipped']}"
    }
    requests.post(webhook_url, json=payload)

def send_email_notification(result):
    msg = MIMEText(f"테스트 실행 완료\n성공: {result['passed']}, 실패: {result['failed']}, 스킵: {result['skipped']}")
    msg["Subject"] = "테스트 실행 결과"
    msg["From"] = "test@example.com"
    msg["To"] = "admin@example.com"
    
    with smtplib.SMTP("smtp.example.com", 587) as server:
        server.starttls()
        server.login("test@example.com", "password")
        server.send_message(msg)
```

## 8. 테스트 실행 주기

### 8.1 CI/CD 파이프라인
GitHub Actions나 GitLab CI를 사용하여 코드 변경시 자동으로 테스트를 실행합니다.

```yaml
# .github/workflows/run_tests.yml
name: Run Tests

on: [push, pull_request]

jobs:
  run_tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: 3.8
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests
        run: pytest tests/ -v --browser=chrome --headless --alluredir=reports/
      - name: Upload Allure report
        uses: actions/upload-artifact@v2
        with:
          name: allure-report
          path: reports/
```

### 8.2 정기 실행
크론탭이나 Jenkins를 사용하여 정기적으로 테스트를 실행합니다.

```bash
# 크론탭 설정 (매일 오전 9시 실행)
0 9 * * * cd /path/to/project && pytest tests/ -v --browser=chrome --headless --alluredir=reports/
```

## 9. 장점
- **시간 절약**: 반복적인 테스트를 자동으로 실행
- **정확성**: 사람의 오류 없이 일관된 테스트
- **보고서**: 시각화된 테스트 결과 보고서
- **CI/CD**: 코드 변경시 자동으로 테스트 실행

## 10. 결론
이 테스트 자동화 계획은 Selenium을 사용하여 웹 브라우저를 자동화하고, Page Object Pattern을 적용하여 유지보수성 높은 테스트 스크립트를 작성합니다. 테스트 데이터 관리를 통해 일관된 테스트를 수행하고, Allure 보고서로 결과를 시각화합니다. CI/CD 파이프라인과 정기 실행을 통해 테스트 과정을 자동화하고, 결과를 Slack·이메일로 통보합니다.
