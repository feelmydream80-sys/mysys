# 데이터 정의 기능 분석 및 시나리오 테스트 계획

## 1. 기능 분석

### 1.1. 주요 기능 목록

| 기능 카테고리 | 기능명 | 설명 |
|-----------------|--------|------|
| **그룹 관리** | 그룹 카드 렌더링 | API에서 데이터를 가져와 그룹별로 카드 형태로 표시 |
| | 그룹 선택 | 카드 클릭으로 그룹을 선택하고 상세 정보 표시 |
| | 그룹 추가 | 새로운 그룹을 추가하는 모달 창 제공 |
| | 그룹 수정 | 선택한 그룹의 정보를 수정하는 모달 창 제공 |
| | 그룹 삭제 | 그룹을 비활성화 처리하는 기능 |
| **상세 관리** | 상세 정보 로드 | 선택한 그룹의 상세 데이터를 표시 |
| | 상세 항목 추가 | 새로운 상세 항목을 추가하는 모달 창 제공 |
| | 상세 항목 수정 | 선택한 상세 항목을 수정하는 모달 창 제공 |
| | 상세 항목 삭제 | 상세 항목을 비활성화 처리하는 기능 |
| **상태 관리** | 선택 항목 활성화 | 선택한 복수 상세 항목을 활성화 |
| | 선택 항목 비활성화 | 선택한 복수 상세 항목을 비활성화 |
| **검색 및 필터** | 그룹 검색 | 검색어로 그룹을 필터링 |
| | 표시 옵션 | 코드+명칭, 명칭만, 삭제그룹만 표시 |
| **UI/UX** | Toast 알림 | 작업 결과를 알려주는 알림 |
| | 모달 창 | 추가/수정/삭제 등의 작업을 위한 모달 |

### 1.2. 데이터 구조

#### tb_con_mst 테이블 구조
- cd_cl: 그룹 코드 (CD100, CD200 등 100의 배수)
- cd: 데이터 코드 (CD101, CD102 등 그룹 내 번호)
- cd_nm: 데이터 명칭
- cd_desc: 활용 목적
- item1~item10: 사용자 정의 필드
- use_yn: 사용 여부 (Y/N)
- update_dt: 수정일시

#### 그룹/상세 구분
- 그룹: cd_cl와 cd가 같은 항목 (예: cd_cl=CD100, cd=CD100)
- 상세: cd_cl과 cd가 다른 항목 (예: cd_cl=CD100, cd=CD101)

### 1.3. API 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| /api/data_definition/groups | GET | 전체 데이터 그룹 정보 가져오기 |
| /api/data_definition/create | POST | 그룹 또는 상세 항목 생성 |
| /api/data_definition/group/:cd | PUT | 그룹 정보 수정 |
| /api/data_definition/group/:cd | DELETE | 그룹 삭제 (use_yn=N) |
| /api/data_definition/detail/:cd | PUT | 상세 항목 수정 |

### 1.4. 주요 로직

#### 전역 상태 관리
- allData: API에서 가져온 전체 데이터를 저장하는 전역 변수
- selectedGroup: 현재 선택된 그룹 정보
- selectedRow: 현재 선택된 상세 행 정보
- isModalOpen: 모달 창 열림 상태
- isInitialized: 초기화 상태

#### 중복 API 호출 방지
- 전역 allData 변수를 사용하여 중복 API 호출 방지
- debounce 함수로 입력시 반복 요청 방지

#### 데이터 동기화
- 모든 데이터 변경 작업 후 allData 재로드
- 그룹 카드와 상세 정보를 다시 렌더링

## 2. 시나리오 테스트 계획

### 2.1. 기본 기능 테스트

- **Test Steps:**
  1. 데이터 정의 탭으로 이동
  2. 페이지 로딩 완료 확인
- **Expected Results:**
  - 그룹 카드가 정상적으로 렌더링
  - 상세 패널이 숨겨져 있음
  - 검색창과 표시 옵션이 표시

#### Test Case 2: 그룹 선택
- **Test Steps:**
  1.任意의 그룹 카드를 클릭
  2. 상세 패널 확인
- **Expected Results:**
  - 선택된 그룹 카드에 하이라이트가 표시
  - 상세 패널에 선택된 그룹의 상세 데이터가 표시
  - 작업 버튼이 활성화

#### Test Case 3: 상세 행 선택

  1. 그룹을 선택
